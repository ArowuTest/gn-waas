# GN-WAAS Staging Deployment Guide
## Vercel (Frontends) + Render (Backend + Database)

**Purpose:** Live testing with seeded demo data  
**Auth:** DEV_MODE=true (no Keycloak needed — use dev-login endpoint)  
**Time to deploy:** ~30 minutes

---

## Architecture on Staging

```
Vercel                          Render
──────────────────────          ──────────────────────────────────────
gn-waas-admin.vercel.app   →   gnwaas-api.onrender.com  (API Gateway)
gn-waas-gwl.vercel.app     →       ↓ internal
gn-waas-authority.vercel.app   gnwaas-sentinel.onrender.com
gn-waas-landing.vercel.app     gnwaas-tariff-engine.onrender.com
                                gnwaas-gra-bridge.onrender.com
                                gnwaas-ocr-service.onrender.com
                                gnwaas-postgres (Render managed DB)
                                gnwaas-redis    (Render managed Redis)
```

## PART 1 — Deploy Backend on Render

### Step 1: Connect repo to Render

1. Go to [render.com](https://render.com) → Sign in
2. **New** → **Blueprint**
3. Connect GitHub → select `ArowuTest/gn-waas`
4. Render detects `render.yaml` automatically → click **Apply**
5. Render creates: 1 PostgreSQL, 1 Redis, 1 API Gateway, 4 private services, 1 migration job

### Step 2: Set the one required secret

After the blueprint creates services, go to:
**Render Dashboard → gnwaas-migrate → Environment**

Add:
```
GNWAAS_APP_PASSWORD = <generate: openssl rand -base64 24>
```

Copy this value — you'll need it in Step 3.

> All other secrets (DB password, Redis password) are auto-generated by Render
> and injected via `fromDatabase` / `fromService` references in render.yaml.

### Step 3: Run migrations + seed data

1. **Render Dashboard → gnwaas-migrate → Manual Run**
2. Watch the logs — should see 33 migrations + 6 seed files complete
3. Takes ~3 minutes

**What gets seeded:**
- 10 districts across Ghana
- 25 demo users (all roles)
- 500 water accounts with realistic billing data
- 3 months of meter readings
- Pre-generated anomaly flags and audit events

### Step 4: Note your API URL

After `gnwaas-api` deploys (green), copy its URL:
```
https://gnwaas-api.onrender.com
```

## PART 2 — Deploy Frontends on Vercel

Deploy each frontend as a **separate Vercel project**.

### Admin Portal

1. [vercel.com](https://vercel.com) → **New Project** → Import `ArowuTest/gn-waas`
2. **Root Directory:** `frontend/admin-portal`
3. **Framework:** Vite (auto-detected)
4. **Environment Variables** (add these):

| Variable | Value |
|---|---|
| `VITE_API_URL` | `https://gnwaas-api.onrender.com` |
| `VITE_DEV_MODE` | `true` |
| `VITE_KEYCLOAK_REALM` | `gnwaas` |
| `VITE_KEYCLOAK_CLIENT_ID` | `gnwaas-admin-portal` |

5. Click **Deploy**

### GWL Portal

1. **New Project** → same repo
2. **Root Directory:** `frontend/gwl-portal`

| `VITE_KEYCLOAK_CLIENT_ID` | `gnwaas-gwl-portal` |

## PART 3 — Wire CORS (connect Vercel → Render)

After all Vercel deployments complete, go to:
**Render Dashboard → gnwaas-api → Environment**

Add:
```
CORS_ALLOWED_ORIGINS = https://gn-waas-admin.vercel.app,https://gn-waas-gwl.vercel.app,https://gn-waas-authority.vercel.app
```

Replace the URLs with your actual Vercel deployment URLs.

Click **Save** — Render redeploys automatically (~2 min).

## PART 4 — Log In and Test

### How authentication works in staging (DEV_MODE=true)

The frontend shows a **"Dev Login"** panel with role buttons. Click any role to log in instantly — no password needed.

Available roles for testing:
| Role | What you can test |
|---|---|
| `SYSTEM_ADMIN` | Full access — all dashboards, user management |
| `GWL_MANAGER` | District dashboards, case management |
| `GWL_ANALYST` | NRW analysis, water balance |
| `GWL_EXECUTIVE` | Executive summary, reports |
| `GRA_OFFICER` | GRA compliance, signed audits |
| `MOF_AUDITOR` | National view, revenue recovery |
| `FIELD_SUPERVISOR` | Field job dispatch, officer tracking |
| `FIELD_OFFICER` | Field jobs (mobile app role — test via API) |

### Direct API login (for mobile app testing)

```bash
curl -X POST https://gnwaas-api.onrender.com/api/v1/auth/dev-login \
  -H "Content-Type: application/json" \
  -d '{"role": "FIELD_OFFICER"}'
```

Returns a JWT token. Use as `Authorization: Bearer <token>` in mobile app.

### Mobile app (APK) for staging

```bash
cd mobile/field-officer-app-flutter
flutter build apk --release \
  --dart-define=API_BASE_URL=https://gnwaas-api.onrender.com/api/v1 \
  --dart-define=KEYCLOAK_URL=https://gnwaas-api.onrender.com \
  --dart-define=KEYCLOAK_REALM=gnwaas
```

The app will use the dev-login flow automatically when `DEV_MODE=true` is detected from the API.

## Troubleshooting

### "Service unavailable" on first load
Render free tier services spin down after 15 minutes of inactivity.
First request takes ~30 seconds to spin up. This is normal for free tier.

### CORS errors in browser
Check that `CORS_ALLOWED_ORIGINS` in Render matches your exact Vercel URL
(including `https://`, no trailing slash).

### Migration failed
Check logs in Render Dashboard → gnwaas-migrate → Logs.
Common issue: `CREATE EXTENSION postgis` fails (PostGIS not available on Render free tier).
This is non-fatal — the migration runner continues. GPS features use lat/lng columns instead.

### Frontend shows blank page
Check browser console for `VITE_API_URL` errors.
Ensure the env var is set in Vercel project settings (not just vercel.json).

### Database connection refused
Render free PostgreSQL takes ~2 minutes to provision.
Wait for the database to show "Available" in Render dashboard before triggering migrations.
